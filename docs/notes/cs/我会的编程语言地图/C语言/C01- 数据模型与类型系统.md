###### 数组的赋值——数组不是“一等公民”
>相关对比：
>[[JS01-数据模型与类型系统#字符串的整体赋值]]

在C语言中，数组是不可以*通过数组名*来进行**整体赋值**的。究其原因是，在C语言的设计中，数组被视作一种**派生类型**，它在内存中占据一块连续的区域，但是语言层面却没有为它定义一个完整的**值语义**。(数组仅仅只是通过首地址和容量来标识，数组名是**首地址的指针常量**)
```c
#include <stdlib.h>
#include <stdio.h>
#include <string.h>

int main(){
    char a[] = "hi";
    char b[2];
    b = a;    //❎ b是一个指针常量，不满足“Left-Value”，会报错。
    char *p = malloc(strlen(a) * sizeof(char));  
    p = a;    //✅ p是一个指针变量，可以被赋值，但此时的赋值是地址的赋值，
			  //   因此实际上的效果是两个指针指向同一块内存区域
    p[1] = 'e';
    printf("%s,%s",a,p); // 结果是he,he，说明会彼此影响。
    return 0;
}
```
不论是`b = a`,还是`p = a`，只要将`数组名a` 当作`Right-Value`进行赋值的时候，其实都只能将数组的指针赋值过去，而不能直接地将数组内容**整体赋值过去（而不受后续修改的彼此影响）**。

不过也有解决办法，就是利用`memcpy`函数，但实际上原理也是循环一个个赋值，因此时间复杂度是`O(n)`。
```c
#include <string.h>

int main(){
    int a[5] = {1,2,3,4,5};
    int b[5];
    b = a;   //❌ b是一个指针常量，不满足“Left-Value”
    memcpy(b, a, sizeof(a));
    printf("%d", b[4]);
    return 0;
}
```

### C的内存管理
C语言是**强类型**、**静态编译**的程序设计语言。
>💡关于“强类型”与“弱类型”的术语澄清：
>一般来说我们对于类型系统有两个划分视角，根据对类型的检查时期可以划分为*动态和静态*；根据转换强度可以划分为*强类型和弱类型*。
>C语言划分为强类型的主要原因是，尽管类似`int a = 3.14`的代码可以正常通过编译（会产生截断，即隐式转换），但是目前的编译器（如clang，gcc）普遍会发出警告。因此我们在这里还是认为C语言是强类型的语言，尽管*大量的隐式转换和指针的任意转换*让其安全性不如Java/C#

**强类型**意味着*不同类型之间的转换需要显式声明，编译器会严格检查类型*，**静态编译**意味着*变量的类型在编译时就确定，运行时不能改变*。
```C
// 先来看一个体现静态编译的例子

#include <stdio.h>

int main(){
    int a;  //没有初始化，仅仅是分配了内存（4个字节）
    printf("%p",(void*)&a); // 打印出分配给变量a的地址
    return 0;
}
// 不知道有没有朋友和我曾经有过相同的困惑，为什么C语言不赋初值的话会出现随机的内容？
// 其实原因就在于C语言分配内存是通过操作系统进行的，虽然分配到的内存此时可能有存储内容（你所看到的随机内容）
// 但是操作系统在给进程/程序创建地址空间的时候判定这一块内存时空闲的（即使它有存储内容）
```
C语言的内存管理是需要在程序中手动进行的，与较高层次的语言（如JavaScript或Python）可能具有GC机制不同，C语言是直接沟通内存的。有这么几种说法：
- 内存管理对C语言这样的语言是**不透明**的，对JS和Python这样的语言是**透明**的。
- 对于JS/Python这样的高级语言，之所以可以在==分配内存和释放内存==时实现隐式进行，本质上也是Python解释器/JS引擎在背后先将JS的源码翻译为C/C++这类更好被翻译为机器码的语言。
- JS/Python这样的高级语言不是不需要内存管理，而是内存管理都由Python解释器/JS引擎代劳了。
>总结：
>C变量的本质是**一块特定大小、特定生命周期的内存区域**，而变量名只是我们访问这块区域的**逻辑句柄**。

| 程序设计语言 | 分配内存 | 读写内存 | 释放内存 |
| ------ | ---- | ---- | ---- |
| C      | 显式   | 显式   | 显式   |
| Python | *隐式* | 显式   | *GC* |
| JS     | *隐式* | 显式   | *GC* |
###### C变量的本质——内存地址的别名
正是因为C语言需要手动管理内存，也就是管理这些变量的*逻辑地址*（即操作系统语境下的虚拟地址，非真正的物理地址），因此除了常见的变量类型以外，需要特地引入一种特殊的变量类型，即**指针变量**。如果从内存管理的角度考察指针变量指向进程地址空间的不同位置，就很好理解指针的作用机制了：
- 指针指向栈区：未加`static`修饰的局部变量、函数参数（形参）、块级变量等
- 指针指向堆区：手动分配内存的变量（通过调用`malloc()`、`calloc`或`realloc`）
- 指针指向静态/全局存储区（Data Segment）：全局变量（定义在函数以外）、静态变量（*包含静态局部变量和静态全局变量*，均以`static`关键字修饰）、常量字符串等
> 🤔静态全局变量 vs 静态局部变量
	总结来说，`static`关键字的作用是限制变量的作用域，如果加在局部变量前，那么就表示这个变量**仅作用于当前函数/块内**；如果加在全局变量前，那么就表示这个变量**仅作用于当前c文件内**。其他C文件无法使用`extern`关键字引用。
	此外，如果一个变量对一个区域没有作用，自然不会有命名冲突。这就是为什么静态变量可以避免命名冲突，虽然这不是它的主要用途。
