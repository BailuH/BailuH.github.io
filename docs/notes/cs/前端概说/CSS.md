# CSS - 层叠样式表
## 核心语法

**选择器和声明块**，声明块内又包含多个*属性-值对*

![image.png](https://bailu-1382509292.cos.ap-beijing.myqcloud.com/obsidian/20260309153555912.png)

### 声明块
在 CSS 中，无论是属性名还是属性值都是对**大小写不敏感**的。属性与值之间以英文冒号 ':' （U+003A COLON）隔开。属性与值前面、后面与两者之间的空白不是必需的，会被自动忽略。

![image.png](https://bailu-1382509292.cos.ap-beijing.myqcloud.com/obsidian/20260309173557478.png)

目前 CSS 中有 [100 多个属性](https://developer.mozilla.org/zh-CN/docs/Web/CSS/Reference) ，并且几乎有无限多个不同的值。并非所有的属性与值的配对都是被允许的，每个属性都定义了它的合法值。如果值对给定的属性而言非法时，声明就会被视为无效的，整个声明就会被 CSS 引擎忽略。

### 选择器
选择器可以根据**作用域**分为两类，**基本选择器和类选择器**。
前者直接写属性名作为选择器，后者需要以`.`标识，例如`.done`，`.i`，只有通过标签的`class`属性才会绑定样式，不会全局应用。
> `#`标识的选择器是**ID选择器**，以`#`开头，例如`#header`、`#nav`。ID在*页面/网页*中应该是*唯一*的，用于标识单个特定元素。

### 完整的选择器分类（根据MDN）
![[CSS Selector Cheat Sheet - Dark.pdf]]

| 选择器类型         | 语法示例                                  | 说明                 |
| ------------- | ------------------------------------- | ------------------ |
| **通配选择器**     | `*`                                   | 选择所有元素             |
| **元素（类型）选择器** | `p`, `h1`, `div`                      | 根据HTML标签名选择        |
| **类选择器**      | `.classname`                          | 选择带有特定class的元素，可复用 |
| **ID选择器**     | `#idname`                             | 选择特定ID的元素，应唯一      |
| **属性选择器**     | `[type="text"]`, `[href]`             | 根据属性或属性值选择         |
| **伪类**        | `:hover`, `:focus`, `:first-child`    | 基于元素状态或位置选择        |
| **伪元素**       | `::before`, `::after`, `::first-line` | 选择元素的特定部分          |
| **组合选择器**     | `>`, `+`, `~`, 空格                     | 后代、子代、相邻兄弟等关系      |
## CSS语句（CSS statement）
CSS的核心语法是前文提及的**选择器与声明块**，二者共同构成了**规则集**（rule-set）。
**规则集**已经占据了样式表的绝大多数内容，但是网页开发者还需要在样式表中传递其他信息，比如*字符集、要导入的外部样式表、字体设置或列表计数器描述等等*。
因此完整的CSS语句除了**规则集**以外，还有一种称之**为@规则**（At-rules）的语句。==这种语句以@开头，后跟一个标识符，并持续到语句结束==。每种由标识符定义的@规则类型可能具有其自身的内部语法和语义。它们用于传达==元数据信息==（如 `@layer` 或 `@import` ）、==条件信息==（如 `@media` 或 `@document` ）或==描述性信息==（如 `@font-face` ）。

此外，CSS语句还有一个*功能分类*——**嵌套语句**（Nested statements），应用于==实现条件化、作用域化的样式控制==。它并不是一个具体的语法类型，它包括了：
- 所有可嵌套的@规则（`@media`、`document`等）
- 以及被嵌套在内的规则集。

## CSS书写在哪？（作者样式表）

### 内联样式
**写在标签内部**，作为一个**属性**存在。
```html
<p style="color: red; font-size: 16px;">这是内联样式</p>
```
### 内部样式
**写在`<head>`标签内部**，作为一个`<style>`**标签**存在。
```html
<head>
  <style>
    p { color: blue; }
  </style>
</head>
```
### 外部样式
**写在`.css`文件内**，作为一个**文件**存在，通过`<link>`引入。
```css
p { color: blue; }
```
```html
<link href="./styles/style.css" rel="stylesheet" />
```

> 😫CSS层叠算法的类型来源
> CSS的声明块可以不同的来源，最完整的分类是包含三类：==用户代理样式表、作者样式表、和用户样式表==。
> 前文所说的三种CSS声明块书写的位置都属于**作者样式表**，是*网页编写者*为网页定制的“主题”。这也是我们大多数时候作为网页开发者会直接接触的样式表。
> 
> 而**用户代理样式表**则是基于浏览器的，会为网页提供一个默认样式。除非**用户代理样式表**中的某个属性包含`!important`标记，提高了其优先级，否则作者样式表会覆盖其基本样式。
> **用户样式表**则指的是网站用户通过*浏览器的直接配置*或者*浏览器插件*等方式指定的样式。

## 层叠算法——决定CSS样式的应用顺序
 [CSS层叠算法](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Cascade/Introduction#complete_cascade_order)
简单来说，浏览器在渲染HTML元素的样式的时候，遵照以下流程：
1. 将页面元素和CSS样式规则关联起来。这个过程是通过**选择器**来完成的。
2. 根据样式的类型来源以及是否有`!important`标记来定**层叠顺序**。
3. 当上述层叠顺序一致时，根据**特异性**（specificity）来决定应用哪一种样式，**特异性**是基于不同种类选择器组成的匹配规则。
4. 如果出现了`@scope`，按照其冲突解决规则进行。
5. 同特异性的情况下，按照**源代码顺序**应用样式，后写的样式覆盖前写的样式。

![[css-cascade-algorithm-flowchart.excalidraw]]

### 选择器匹配
浏览器首先通过**选择器**将 CSS 规则与 HTML 元素关联。只有匹配目标元素的选择器才会进入后续比较。可以从两个关键的视角看待这个过程：
1. 一个元素可能匹配多条规则，例如：
```css
#nav .item { color: blue; }  /* 匹配 id="nav" 内的 class="item" */
.item { color: red; }        /* 也匹配 class="item" */
```
2. HTML元素要应用某种CSS样式，必须要属于某些 `@media` 规则（At-rules）。
```css
@media [media-type] (condition) {
	selector {
		property1 : value1;
		property2 : value2;
		}
	}
```
[CSS与HTML元素无法匹配的示例](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Cascade/Introduction#basic_example)

### 层叠顺序——在反思中前进
层叠顺序是层叠算法*最复杂、最有历史包袱*的一部分，影响因素不仅包括上文提过的==样式来源==和 `!important`==标记== ，还涉及 `@layer` ==层叠层==。对于大多数网页编写者而言，甚至还有作者样式表的==内联样式==需要考虑。
- 样式来源：作者样式表 > 用户样式表 > 用户代理样式表（浏览器默认样式表）
- 层叠层：层与层之间若无声明，则默认后面的层的声明优先于前面的层的声明，层内的声明优先级总是小于层外的普通样式。
```css
/* @layer 的三种主要语法形式 */
/* 声明层顺序（无花括号）
	仅声明层的优先级顺序，不写入任何样式
	一般都会在使用层之前先声明顺序，否则就按首次出现顺序*/
@layer reset, base, components, utilities;

/* 定义层内容（有花括号）
	可以多次追加，同一层的样式会合并*/
@layer base {
  body { font-family: system-ui; }
  h1 { font-size: 2rem; }
}

/* 匿名层
	可以配合导入使用，也可以声明匿名层*/
@import url('normalize.css') layer(reset);
@import url('bootstrap.css') layer(framework);

@layer {
  p {
    margin-block: 1rem;
  }
}
```
- `!important` 标记：`!important`标记可以逆转上述的两条一般规则，即==作者样式表 > 用户样式表 > 用户代理样式表==和==层内的声明优先级总是小于层外的普通样式==。
- 内联样式：在作者样式表中，内联样式优先级大于未分层的样式声明。

### 特异性计算——看似美好，实则隐患
> 🤔特异性的引入带来的“*特异性战争*”
> 绝大多数情况下，网页开发者直接操作的都是作者样式表，更具体地，往往是其中的==内部样式或外部样式==。（因为内联样式得写在网页本身上）
> 这导致大多数情况下决定CSS应用顺序的情形其实都取决于选择器的==特异性==。当你引入一个第三方CSS库（如Bootstrap），发现它的样式特异性过高，为了应用自己写的样式，我们被迫卷入“*特异性战争*”——不断提高自己写的样式特异性（通过增加类名，使用ID选择器或者干脆直接使用 `!important` 标记）。这至少可能会导致以下问题：
> 1. **选择器臃肿**：类似 `.header .nav .item a.link.active` 这样复杂的选择器难以阅读和维护。
> 2. **CSS样式与HTML元素耦合**：CSS 依赖特定 DOM 层级，一改 HTML 就崩。
> 3. **`!important` 泛滥**：一旦使用，后续只能用更多 `!important` 覆盖。

> ✅ *Layers* 如何解决战争？
> 通过在确定**层叠顺序**环节设计==来源层（Layers）==，既保留了样式复用的便捷性，也给予了CSS开发者根据项目本身情况决定层叠顺序的灵活性。
> 默认认为来源层声明优先级低于层外的普通声明块，这就允许CSS编写者只需要编写很低的特异性声明块，就能覆盖其他第三方CSS样式（通过将其导入为层）。

当**层叠顺序相同**时（例如两条规则都来自同一个 `@layer` 或未分层样式），浏览器进入**特异性**（Specificity）计算。
现代 CSS 使用**三元组** `(ID, CLASS, TYPE)` 计算特异性：

| 选择器组成部分     | 对应列 | 示例                            | 贡献值   |
| ----------- | --- | ----------------------------- | ----- |
| **ID 选择器**  | 第1位 | `#header`                     | 1-0-0 |
| **类/属性/伪类** | 第2位 | `.active`, `[type]`, `:hover` | 0-1-0 |
| **元素/伪元素**  | 第3位 | `div`, `::before`             | 0-0-1 |
| **通配符 `*`** | 无   | `*`                           | 0-0-0 |
计算示例：
```css
#nav               /*  → (1,0,0)   1个ID */
.menu li:hover     /*  → (0,2,1)   2个类/伪类 + 1个元素 */
a[href]            /*  → (0,1,1)   1个属性 + 1个元素 */
```
**比较规则**：从左到右逐位比较，第一位大的胜出，不看后面位。例如 `(0,2,0)` > `(0,1,99)`。
**特殊规则**：内联样式的特异性可以被看作为 `(1,0,0,0)`（四位），高于任何特异性，==不需要参与特异性比较，因为其在层叠顺序阶段就已胜出。==

### 作用域邻近性——@scope是如何改变源代码顺序的
当**特异性相同**时，如果存在 `@scope` 规则，浏览器会评估**作用域临近性（Scope Proximity）**。
选择器匹配的**作用域根元素**（scoping root）在 DOM 中距离目标元素**越近**，优先级越高。
[MDN官方给的一个解决@scope的示例](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/At-rules/@scope#how_scope_conflicts_are_resolved)

### 源代码顺序
当以上所有条件都相同时，最后声明的样式（**Last Wins**）胜出。
值得强调的是，外部样式的导入，声明的位置认为在导入语句的位置。
