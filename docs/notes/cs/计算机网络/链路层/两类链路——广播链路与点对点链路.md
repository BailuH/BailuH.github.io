
## 网络节点的连接方式

> ❓一个子网中的若干节点是如何连接到一起的

**节点**在链路层的语境之下指的大致有这么几类设备：*主机、路由器、链路层交换机、WiFi接入点*。
各节点通过所谓的**链路**连接在一起，而链路可以分为两类（*广播链路*和*点对点链路*），二这两种链路各自都有几种物理拓扑结构，值得强调的是，==物理的拓扑形态并不影响链路本身的性质与逻辑特征==。
- 点对点链路：
	- 将**两个**节点通过某种介质直接连接在一起，这种方式一般会形成**网状拓扑**结构。
	- 通过交换机（链路层交换机）将节点都接入到交换机上，可以视作是每两个节点之间有一个虚拟点对点链路连接。换句话说，交换机对于节点是透明的。这种方式会形成**星形拓扑**结构。
- 多点连接
	- 将若干节点通过共享型介质（如同轴电缆）连接，这种方式形成的是**总线型拓扑**（**bus**）结构，现今已被淘汰。
	- 通过集线器（Hub）将若干节点都连接到Hub上，本质上是用双绞线模拟了总线功能，形成的是**星型拓扑**结构
	- 通过交换机（需要特殊配置）将节点都接入到交换机上，这时候交换机可以处理广播帧，形成的也是**星型拓扑结构**
	- 环型拓扑类型（*物理上的实现？*）
	- *无线广播*，介质就是空气，没有物理线缆，所有节点共享同一频段的电磁波。

| 链路特征     | 广播链路                              | 点对点链路               |
| -------- | --------------------------------- | ------------------- |
| **节点数量** | 技术上可连接**3个及以上**节点                 | **严格仅2个**节点         |
| **地址机制** | 需要**MAC地址**区分多节点                  | **无需MAC**，或仅用协议字段标识 |
| **广播能力** | 支持广播（帧可到达所有节点）                    | **不支持广播**（帧只能到对端）   |
| **典型协议** | 以太网(Ethernet)、Wi-Fi、令牌环【这里指的是协议族】 | PPP、HDLC            |
| **子网特征** | 通常/24或更大（容纳多主机）                   | 通常/30或/31（仅2个IP）    |
以太网随着技术的发展，已经成为了一个功能强大且复杂的网络，不仅具有广播链路的特性，同时也具备一些点对点链路的特性（例如使用链路层交换机工作）。
一般来说，点对点链路目前主要用于*广域网*，例如跨国海底电缆铺设；而以太网则是主要的*局域网*技术。
