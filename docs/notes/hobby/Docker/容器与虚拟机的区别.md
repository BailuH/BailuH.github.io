**容器**：一个用于运行应用程序的隔离环境

**虚拟机**：一种对于计算机物理硬件的*抽象*（我们在操作系统中很熟悉这个概念）
- 在一台物理机上可以实现多个虚拟机，这要借助于*虚拟机监控程序*（*Hypervisor*）来实现，常见的*Hypervisor*有==Virtual Box、VMware、Hyper-v（Windows Only）==。

![image.png](https://bailu-1382509292.cos.ap-beijing.myqcloud.com/obsidian/20260330171206219.png)
- 

二者似乎都能完成我们需要隔离环境以运行不同应用程序的目标，但是虚拟机的方式有几个问题，这也是容器技术应运而生的原因：
1. 每台虚拟机都需要一个完整的操作系统副本，并且还得获得授权，这会导致两个最直接的问题，*启动慢、资源消耗大*。
2. 每个虚拟机都需要独立进行配置，配置压力大。

容器技术相对比之下就有很多好处：
1. 能够完成隔离不同应用程序的基本需求
2. 更加轻量级
3. 仅使用一个主机操作系统，减少配置压力
4. 启动快
5. 硬件资源占用少
