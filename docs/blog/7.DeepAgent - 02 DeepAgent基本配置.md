# 用 TypeScript 打造 Deep Agent：环境搭建与 CLI 实战
## 从JS到TS：环境准备

 TypeScript 是 JavaScript 的超集，简单来说就是**给 JS 加上了类型系统**。你写代码时声明变量类型（如 `let name: string`），TS 在编译阶段就能帮你发现类型错误，而不是等到运行时崩溃。它完全兼容 JS，编译后生成纯 JS 代码，所以任何能跑 JS 的地方都能跑 TS。核心好处是：**提前发现 bug、更好的代码提示、重构更安全**，特别适合大型项目协作。

从**JS开发流程迁移到TS**，需要做以下工作：
1. 除非你青睐全局安装，否则在当前项目下运行`npm install -D typescript`，这个命令会在当前项目`node-module`目录下安装TS的编译器，这是必需的。
2. 这一步是可选的，不过加上的话IDE支持类型检查会更好，运行`npm install -D @types/node`，这个命令会在当前项目`node-module`目录下添加类型支持。

然后是**项目目录管理**，这里仅是分享我的做法，不是唯一方法：
1. 进入项目目录，敲`npx tsc --init`，这个命令会调用你刚刚下载好的TS编译器，生成一个编译配置文件`tsconfig.json`，这里面可以配置TS编译器的行为，比较重要的参数我做一个表格放在后面，完整的信息可以访问官方文档[TypeScript：TSConfig 参考文档——每个 TSConfig 选项的文档 --- TypeScript: TSConfig Reference - Docs on every TSConfig option](https://www.typescriptlang.org/tsconfig/)。我这里只修改了`rootDir`和`outDir`的配置，其他的保持默认值。做完这一步，你的项目目录大致应该长这样。
```
project/
├── node_modules   # 模块
├── src/           # 放 .ts 源文件
│   └── index.ts
├── dist/          # 编译后的 .js 文件（自动生成）
├── package.json   # 包配置文件
└── tsconfig.json
```
2. 之后就可以正常安装依赖了，如果是一个别人的项目就`npm install`，我这里是新项目，以安装LangChain的DeepAgent为例，敲`npm install deepagents langchain @langchain/core @langchain/tavily`

然后是**运行TS文件**的方法，有两种常用的方法：
1. 传统老辈子的方法：敲`npx tsc && node dist/index.js`，这个命令会先按照`tsconfig.json`编译`src`下的所有`.ts`源文件，并产生对应的`.js`文件到`dist`目录下，再用node.js引擎运行`.js`文件。
2. 基于`tsx`转译器的方法：首先得先下载`tsx`，敲`npm install -D tsx`。然后再敲`npx tsx src/index.ts`，调用`tsx`直接对`.ts`源文件进行转译为`.js`代码到内存中，再由node引擎执行。这个过程不会产生`.js`中间文件，但相应地也不会触发编译的类型检查（不过IDE的`@types`类型检查不受影响）

两种方法都比较常用，可以同时使用。然而老辈子方法可能会带来一个这样的问题，就是如果某个`.ts`源文件被删除或者重命名了，那么编译的时候就不会覆盖原本的`.js` 文件，可能产生冗余的文件，虽然无伤大雅，但是最好还是手动管理一下。
方法是这样的：
1. 在`package.json`中添加这样一个配置项：
```json
{
    "scripts": {
    "build": "rm -rf dist && tsc"
  }
}
```
这样你要用老辈子方法的时候就先敲`npm run build`，会在当前目录下执行Linux/MacOS终端的命令`rm -rf dist && tsc`，这段命令的意思是递归强制删除当前目录下的`dist`目录（与所有文件），再编译所有`.ts`源代码。如果你是WIN系统的话就把这段命令换成WIN的对等物就好。

## 安装DeepAgent-CLI并连接到硅基流动

硅基流动的API是兼容OpenAI格式的，所以我们可以设置环境变量来覆盖`deepagents-cli`的模型配置，关键的信息包括：*BaseURL、API_Key、Models*，设置环境变量或者写进终端（Bash/Zsh）的配置文件：
```bash
export OPENAI_BASE_URL='https://api.siliconflow.cn/v1'
export OPENAI_API_KEY='你的API密钥'
```
然后下载`deepagents-cli[openai]`，官方推荐的安装方法是通过`uv`全局安装，其实也可以直接`pip`安装。
```shell
uv tool install 'deepagents-cli[openai]'
```
安装完成后敲`deepagents`先启动一次，然后就可以在系统中创建一个文件目录`.deepagents`，一定要找到这个目录，是后续配置我们agent的关键。

在`.deepagents`目录下创建一个新的配置文件`config.toml`，写入以下内容：
```toml
[models.providers.openai]
base_url = "https://api.siliconflow.cn/v1"
api_key_env = "OPENAI_API_KEY"
models = [
    "Pro/moonshotai/Kimi-K2.5",
    "Pro/MiniMaxAI/MiniMax-M2.5",
    "Pro/zai-org/GLM-5",
]
```
目前我只配置了三个模型，还可以进行扩展。不过要注意模型名称必须得符合硅基流动的模型名称，最稳妥的办法就是在终端敲以下命令，要注意将`<token>`替换为你的API_Key。
```shell
curl --request GET \
  --url https://api.siliconflow.cn/v1/models \
  --header 'Authorization: Bearer <token>'
```

做完这些事情之后这个CLI基本已经可以使用了，支持读写文件、执行脚本、记忆等基础功能。

## 附录
### 一、`tsconfig.json`重要参数

| 参数名                                | 说明                                     | 常用值/默认值                                                     |
| ---------------------------------- | -------------------------------------- | ----------------------------------------------------------- |
| `target`                           | 编译目标 JavaScript 版本                     | `"ES5"`, `"ES6"`, `"ES2020"`, `"ESNext"`                    |
| `module`                           | 模块系统                                   | `"CommonJS"`, `"ES6"`, `"ESNext"`, `"Node16"`, `"NodeNext"` |
| `strict`                           | 启用所有严格类型检查（总开关）                        | `true` / `false`                                            |
| `noImplicitAny`                    | 禁止隐式 `any` 类型（`strict` 子集）             | `true` / `false`                                            |
| `strictNullChecks`                 | 严格检查 `null` 和 `undefined`（`strict` 子集） | `true` / `false`                                            |
| `esModuleInterop`                  | 启用 ES 模块与 CommonJS 的互操作性               | `true` / `false`                                            |
| `skipLibCheck`                     | 跳过声明文件（`.d.ts`）的类型检查                   | `true` / `false`                                            |
| `forceConsistentCasingInFileNames` | 强制文件名大小写一致                             | `true` / `false`                                            |
| `outDir`                           | 编译输出目录                                 | `"./dist"`, `"./build"`                                     |
| `rootDir`                          | 源代码根目录                                 | `"./src"`                                                   |
| `declaration`                      | 生成 `.d.ts` 声明文件                        | `true` / `false`                                            |
| `sourceMap`                        | 生成 source map 文件                       | `true` / `false`                                            |
| `noEmit`                           | 不输出文件（仅类型检查）                           | `true` / `false`                                            |
| `jsx`                              | JSX 转换模式                               | `"react"`, `"react-jsx"`, `"preserve"`                      |
| `lib`                              | 包含的 API 库                              | `["ES2020", "DOM"]`                                         |
| `moduleResolution`                 | 模块解析策略                                 | `"node"`, `"Node16"`, `"NodeNext"`, `"bundler"`             |
| `resolveJsonModule`                | 允许导入 JSON 文件                           | `true` / `false`                                            |
| `allowJs`                          | 允许编译 JavaScript 文件                     | `true` / `false`                                            |
| `checkJs`                          | 检查 JavaScript 文件中的错误                   | `true` / `false`                                            |
| `paths`                            | 路径别名映射                                 | `{ "@/*": ["./src/*"] }`                                    |
| `baseUrl`                          | 解析非相对模块的基目录                            | `"./"`                                                      |
| `include`                          | 包含的文件/目录（顶层配置）                         | `["src/**/*"]`                                              |
| `exclude`                          | 排除的文件/目录（顶层配置）                         | `["node_modules"]`                                          |
| `extends`                          | 继承另一个 tsconfig 配置（顶层配置）                | `"./base.json"`                                             |
