# 我准备放弃Claude Code，用LangChain手搓一个自己的Obsidian画图搭子
> 家人们好久不见，最近很久没有更新了，这两天一直在学JavaScript，也是加入了JS这个权威的圈子了。最近不是春节嘛，也是用假期时间零零散散地把JS的学习给收尾了，于是想要做一个小项目来巩固自己的学习成果，于是就有了这次更新——用LangChain手搓一个自己的Obsidian画图搭子。

不过话得先说在前面，我在写下这篇文章的时候，还没开始写任何一行代码，甚至也还没仔细去考虑技术实现，只是大体有个方向，我准备将这个小项目做成一个小系列，这篇更新算是一个预告文，~~想看技术内容的同学可以划走了~~。

然后再说一下这个项目的背景吧，还是因为我这两天在学JS的缘故，我现在已经完全拥抱了用Obsidian做笔记的方式，其中我经常用到里面的==Excalidraw插件==来为某一些关键知识点做可视化。就像这样：

![image.png](https://bailu-1382509292.cos.ap-beijing.myqcloud.com/obsidian/20260221003742853.png)

Excalidraw的手绘插画风格真的是深得我心，好看又实用。唯一的**缺点**就是——**太费时间了**。于是我一直都尝试着用Github上的一个项目来辅助我进行画图，这里贴上原项目链接：[axtonliu/axton-obsidian-visual-skills: Visual Skills Pack for Obsidian: generate Canvas, Excalidraw, and Mermaid diagrams from text with Claude Code](https://github.com/axtonliu/axton-obsidian-visual-skills)

实际上我是在b站刷到这个视频才看到这个项目的，如果不想直接看项目也可以看视频：[【Obsidian 官方开源 Claude Skills 后，我把「可视化三件套」开源了（Canvas/Excalidraw/Mermaid） | 回到Axton】](https://www.bilibili.com/video/BV1gu6RBTETL?vd_source=b51eb5e008b294966f12953b4f210acf)

其实两个都可以不用看，无非就是借助Claude Code强大的Agent能力，搭配前段时间很火的“Skills”，让AI生成一定格式的MarkDown文件，然后就可以用Obsidian打开进一步编辑或者嵌入笔记。
但是目前这个项目有几个**痛点**亟待解决：
- 首先是刚需一个好用的**AI-cli产品**，至少要支持“Skills”功能，不然对于输出的结构化没有保证，仅用提示词工程搭配网页端AI产品这样的方案显得不够“原生”，不够便捷。
- 如果是用Claude Code，由于Anthropic对中国的封锁，对于大多数人而言比较麻烦，Sonnet、Opus系列的模型也比较贵，目前像Claude Code这类变成产品国内平替只有几家，**模型选择**不够多。
- 大多数时候我目前使用的“Claude Code + Kimi-K2.5 API调用“方法，一个是很慢，而是准确率似乎也不高。（甚至偶尔还是有格式问题），究其原因我觉得还是因为画图（尤其是有参考图画图）本质上**并不属于强推理任务**，Claude Code内部有太多针对编码这样的强推理任务的提示词优化了，有时反而适得其反。

我仔细看了看LangChain提出的”Deep-Agent“架构，实际上能看出来设计思路和Claude Code大同小异，只是技术实现上还是建立在自家的LangChain的**ReAct代理架构**之上。但是现在的LangChain用另一个术语**Agent Harness**以表示和前端框架ReAct做区分。

![image.png](https://bailu-1382509292.cos.ap-beijing.myqcloud.com/obsidian/20260221100412525.png)

更令人兴奋的是，Deep Agent还预构建了一个CLI工具，这使得我们的开发难度大大降低，基本上只需要做一个简单自定义配置就可以立刻跑起来了。总结一下我们目前的准备，如果有朋友想长期追更可以参考：
- *Deep Agent CLI*：一个基于虚拟文件系统的终端代理
- *硅基流动的众多大模型API*：可以通过LangChain与构建的聊天代理集成
- *Obsidian与Excalidraw插件*：不用多说，我的电子笔记主力战场。
- *JS的语言基础*：LangChain一直是TS和Python齐头并进的，我之前大多数时候都用的Python，这次出于锻炼自己JS功力，准备使用TS作为自己的编程语言。
